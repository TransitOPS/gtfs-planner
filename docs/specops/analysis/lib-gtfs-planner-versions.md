## 1. Summary

The GTFS Versions context manages named GTFS dataset versions within each organization. It provides version CRUD (create, read, update), version listing, dropdown data for UI select components, and latest-version lookup — all scoped to an organization boundary. Versions are created automatically on organization creation (a "First Version" default) and act as a foreign key anchor throughout the system: stops, routes, pathways, validation runs, walkability tests, and other GTFS entities all reference a `gtfs_version_id`.

The context has two source files: the context module (`lib/gtfs_planner/versions.ex`) and the Ecto schema (`lib/gtfs_planner/versions/gtfs_version.ex`). There is no pubsub broadcasting from this context; mutating operations are invoked by upstream contexts (organizations, accounts) that handle their own broadcasting.

**Version switching** is not managed by this context — it does not track "current" or "active" version state. The web layer holds a `current_gtfs_version` assign typically derived from `GtfsPlanner.Versions.get_latest_gtfs_version/1` or from user session selection.

## 4. Data Flow

### 4.1 Version Creation Flow

```
Caller (Organizations, Accounts)
  → Versions.create_default_version(org_id)
    → Versions.create_gtfs_version(org_id, %{name: "First Version"})
      → %GtfsVersion{organization_id: org_id}
        |> GtfsVersion.changeset(attrs)
          |> cast([:name])
          |> trim_string_fields()       # String.trim/1 on all :string fields
          |> validate_required([:name])
          |> validate_length(:name, min: 1, max: 255)
          |> unsafe_validate_unique([:name, :organization_id], Repo)
          |> unique_constraint(:name, name: :gtfs_versions_organization_id_name_index)
        |> Repo.insert()
```

**Key design decisions**:
- `organization_id` is never in the cast list — set programmatically on the struct before changeset application
- `trim_string_fields/0` runs before `validate_required/1` — so whitespace-only names are trimmed to `""` then fail "can't be blank"
- `unsafe_validate_unique` provides in-memory check for better error messages; `unique_constraint` provides DB-level enforcement with a named constraint
- Organization creation wraps version creation in a `Repo.transaction` — if version creation fails, the organization insert is rolled back

### 4.2 Version Switching

The Versions context does NOT manage version switching. The "current" version for a user session or request is determined entirely by the web layer:

1. On initial access to a GTFS-scoped route, the LiveView or plug calls `Versions.get_latest_gtfs_version(organization_id)` to default to the most recent version
2. A version dropdown (fed by `list_gtfs_versions_for_dropdown/1`) allows the user to manually switch
3. The selected version ID is stored as `current_gtfs_version` (or `current_gtfs_version_id`) in the socket assigns
4. All downstream GTFS queries pass the `gtfs_version_id` explicitly to scoped query functions

There is no `current_version_id` column on organizations or users — the version is purely session/request scoped.

### 4.3 Version Assignment to Dependent Entities

Whenever GTFS data is imported (via mix tasks: `import_stops`, `import_pathways`, `import_levels`), the importing process receives a `gtfs_version_id` as a parameter. The version ID is stored as a foreign key on each imported record (stops, routes, pathways, levels, etc.). This means:
- Each GTFS dataset import creates data under a specific version
- Multiple versions can coexist — each with independent stop/route/pathway data
- Version identity is the partition key for GTFS data queries

## 5. Business Rules & Validation

### 5.1 Name Validation (on `GtfsVersion.changeset/2`)

| Rule | Implementation | Error Message |
|------|---------------|---------------|
| Name is required | `validate_required([:name])` | `"can't be blank"` |
| Name min 1 character | `validate_length(:name, min: 1)` | standard Ecto message |
| Name max 255 characters | `validate_length(:name, max: 255)` | `"should be at most 255 character(s)"` |
| Name uniqueness per org | `unique_constraint(:name, name: :gtfs_versions_organization_id_name_index)` | `"A version with this name already exists"` |
| Whitespace trim | `trim_string_fields/0` (before `validate_required`) | — |

### 5.2 Whitespace Trimming Behavior

All string fields on the changeset are trimmed via `GtfsPlanner.ChangesetHelpers.trim_string_fields/1`, which calls `String.trim/1` on every field of type `:string`. Because trimming runs before `validate_required`:
- A name of `"   "` (whitespace only) becomes `""` → fails "can't be blank"
- A name of `"  Hello  "` becomes `"Hello"` → stored trimmed
- A name of `"  Dup  "` matches existing `"Dup"` → duplicate error

### 5.3 Uniqueness Scope

Version names are unique **per organization**, not globally. Two organizations can each have a version named "Spring 2024". The DB-level unique index `gtfs_versions_organization_id_name_index` enforces this.

### 5.4 No Deletion

The Versions context has no `delete_gtfs_version/1` function. Versions cannot be deleted individually — they are cascade-deleted only when their parent organization is deleted (`on_delete: :delete_all`). This is intentional since versions anchor large volumes of imported GTFS data.

## 6. Error Handling

### 6.1 Changeset Errors

All mutating functions return `{:error, %Ecto.Changeset{}}` on validation failure. The changeset carries per-field error messages. Callers are expected to check the error tuple or use `Ecto.Changeset.traverse_errors/2`.

### 6.2 DB Constraint Errors

If `unsafe_validate_unique` passes (due to race condition) but the DB unique constraint catches the duplicate, `unique_constraint/3` converts the DB error into a changeset error with message `"A version with this name already exists"`. The constraint name `gtfs_versions_organization_id_name_index` is explicitly passed to avoid relying on Ecto's constraint name inference.

### 6.3 Not Found

- `get_gtfs_version/1` returns `nil` for missing IDs
- `get_gtfs_version!/1` raises `Ecto.NoResultsError` for missing IDs
- `get_latest_gtfs_version/1` returns `{:error, :no_versions}` when an organization has no versions
