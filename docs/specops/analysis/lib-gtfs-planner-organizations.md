## 1. Summary

The Organizations context is the multi-tenancy backbone for GtfsPlanner. It manages tenant organization CRUD, API key lifecycle (generation, hashing, serialization, verification), user-organization membership with role-based access control, member activation/deactivation (soft-delete), default GTFS version seeding on org creation, and PubSub broadcasting for all mutating operations.

The context integrates with `GtfsPlanner.Accounts`, `GtfsPlanner.Versions`, `GtfsPlanner.Authorization.Roles`, and `Phoenix.PubSub`. It is consumed by web-layer plugs (`AssignApiOrganization`, `AssignOrganization`), admin LiveViews (`Admin.OrganizationsLive`), and the companion API pipeline.

**Token Format**: `GtfsPlanner.V{version}.{base32_encoded(id || secret)}`

Token is a 3-part dot-separated string:
1. Prefix: `"GtfsPlanner"`
2. Version: `"V1"`
3. Payload: Base32-encoded (lowercase, no padding) concatenation of 16-byte UUID dump + 32-byte random secret

**Token Construction**: `serialize_token/3` in `lib/gtfs_planner/organizations/api_key.ex:54-63`

**Hash Algorithm**: SHA-512 of colon-delimited concat: `"#{api_key_id}:#{version}:#{organization_id}:#{secret}"`

**Verification** (`verify_token/2` in `api_key.ex:71-89`):
1. Split token by `.` into 3 parts
2. Verify prefix matches `"GtfsPlanner"` and version is `"V1"`
3. Decode Base32 payload
4. Extract first 16 bytes as UUID, remaining 32 bytes as secret
5. Look up API key by UUID from DB
6. Recompute SHA-512 hash
7. Constant-time comparison via `Plug.Crypto.secure_compare/2`

**Return values**:
- `{:ok, %ApiKey{}}` on valid token
- `{:error, :invalid}` on any mismatch, malformed token, or missing record

## 4. Data Flow

### 4.1 Organization Creation Flow

```
create_organization(attrs)
  └─ Repo.transaction
       ├─ insert_organization(attrs)
       │    └─ Organization.changeset → cast [:alias, :name]
       │        ├─ trim_string_fields (whitespace)
       │        ├─ normalize_alias (lowercase, strip, hyphenate)
       │        ├─ validate_required [:alias, :name]
       │        ├─ validate_length (max 255 each)
       │        └─ unique_constraint :alias
       └─ Versions.create_default_version(org.id)
            └─ create_gtfs_version(org.id, %{name: "First Version"})
  └─ broadcast {:organizations, :created}
```

If either step fails, the entire transaction is rolled back. The changeset from the failed step is passed to `Repo.rollback/1` and returned as `{:error, changeset}`.

### 4.2 API Key Creation Flow

```
create_api_key(organization_id, attrs)
  ├─ Build changeset: %ApiKey{organization_id: org_id} |> ApiKey.changeset(attrs) |> validate_required(:organization_id)
  ├─ ApiKey.build_hashed_token(org_id, changeset)
  │    ├─ Generate UUID → api_key_id
  │    ├─ Generate 32 random bytes → secret
  │    ├─ hash_api_key(id, version=1, org_id, secret) → SHA-512
  │    ├─ serialize_token(version=1, api_key_id, secret) → "GtfsPlanner.V1.{base32}"
  │    └─ cast id, version, secret_hash into changeset
  └─ Repo.insert(updated_changeset)
       └─ Returns {:ok, {api_key, token}} or {:error, changeset}
```

**Important**: `ApiKey.id` has `autogenerate: false` — the ID is generated in `build_hashed_token/2` before insertion. This ties the token payload to the database record ID.

### 4.3 API Key Verification Flow

```
get_api_key_by_token(token)
  └─ ApiKey.verify_token(token, Repo)
       ├─ Split by "." → [prefix, version, encoded]
       ├─ Match prefix "GtfsPlanner", version "V1"
       ├─ Base32 decode encoded → <<id::16-bytes, secret::32-bytes>>
       ├─ Ecto.UUID.cast!(id) → uuid
       ├─ Repo.get(ApiKey, uuid)
       ├─ Recompute hash(api_key_id, 1, api_key.organization_id, secret)
       └─ Plug.Crypto.secure_compare(recomputed, stored_hash)
```

### 4.4 Organization Resolution for API Requests

```
Plug: AssignApiOrganization (lib/gtfs_planner_web/plugs/assign_api_organization.ex)
  ├─ Requires :current_user on conn (set by VerifyApiSession)
  ├─ Gets active memberships via Accounts.list_user_org_memberships(user_id)
  │
  ├─ IF X-Organization-Id header present:
  │    ├─ Validate UUID format → 400 if malformed
  │    ├─ Check user membership → 403 if forbidden
  │    └─ assign :current_organization_id
  │
  └─ IF no header:
       ├─ 1 membership → auto-assign org_id
       ├─ 0 memberships → 403 (code: "no_organization")
       └─ 2+ memberships → 403 (code: "organization_required", with available IDs list)
```
